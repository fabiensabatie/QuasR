DOCUMENTATION FOR QUASR FUNCTIONS

function ft_insert_user(cl_user $user) | user_model.php :

The function inserts the user (of type cl_user) into the mongo users collection. It then returns the inserted object.

function ft_check_user_integrity(cl_user $user) | user_model.php :

The function checks that none of the properties of the users are NULL. Example : if $users->name happened to be equal to NULL, the function will return false, if all properties have been assigned a value, the function returns true.

function ft_var_dump($var) | basics.php :

Does a simple var_dump($var), but adds the "pre" tags to make it more readable.

function ft_echo(string $string, string $color) | basics.php :

Does a simple echo, but allows to parse in an HTML color to apply to the string.

function ft_check_entries(array $_POST) | ft_register.php :

The function checks the entries from the registration form, and checks if evey field from the main_info array (of the cl_user class) has been filled in the form.

function ft_check_user(cl_user $user) | ft_register.php :

The function checks if the username is free and if the email has not already been used. If it is case, a message indicates where the issue is, and the function returns false. Else, it returns true.

function ft_form_is_valid(array $_POST) | ft_login.php :

The function checks if the login and password fields in the login form have been correctly filled.

function ft_set_session_variables() | ft_login.php :

The function sets the following session's variables are set : (bool)isConnected, (string)user_id, (string)user_first_name, (string)username.

function ft_compare_passwords(string $password, array $user) | ft_login.php :

The function compares the hash from database with the password given by the user. If the password matches the hash, the ft_set_session_variables() function is called, a success message is printed, and true returned. Else, a failure message is printed and false is returned.

function ft_login(array $_POST) | ft_login.php :

The function tries to find a username matching the login provided by the user. If no username is found, the function tries to find a matching email address. If none are found, then it prints an error message and returns false. If a username is found, the function calls ft_compare_passwords() and returns its result. The same happens if an email address is found. If a username was found, the email checks are skipped.
